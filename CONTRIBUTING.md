# Contributing

JARVIS is currently in product-definition and early architecture development.

Before opening an implementation pull request:

1. Read `docs/PRD.md` and `AGENTS.md`.
2. Keep the change focused on one approved vertical slice.
3. Include tests for new state transitions or persistence behavior.
4. Document new dependencies and their licenses.
5. Record directly adapted code in `docs/DESIGN_REFERENCES.md`.
6. Do not add provider-specific assumptions to the core domain model.
